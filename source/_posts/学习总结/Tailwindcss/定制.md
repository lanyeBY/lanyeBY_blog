### Content
`tailwindcss` 的工作原理是扫描所有HTML、JavaScript组件和任何其他模板文件中的类名，然后为这些样式生成所有相应的CSS。
`content` 属性就是用来配置你需要被 `tailwindcss` 转换的文件列表。

```javascript
module.exports = {
  content: [
    './pages/**/*.{html,js}',
    './components/**/*.{html,js}',
    './index.html',
  ],
  // ...
}
```

需要注意：路径是相对于项目根，而不是相对于 `tailwind.config.js` 文件。
如果确实需要使路径是相对于配置文件（`tailwind.config.js`），可以使用属性 `relative`：

```javascript
module.exports = {
  content: {
    relative: true,
    files: [
      './pages/**/*.{html,js}', './components/**/*.{html,js}',
    ],
  },
  // ...
}
```

其他文件名称的规则如下：

- `*`：匹配除斜杠和隐藏文件以外的任何内容
- `**`：匹配零个或多个目录
- `{}`：{}之间使用逗号分隔值来匹配选项列表

#### 动态类名
请避免使用类名拼接的方式，如果存在逻辑，应该保证字符串完整存在：

```html
<!-- 不生效 -->
<div class="text-{{ error ? 'red' : 'green' }}-600"></div>

<!-- 生效 -->
<div class="{{ error ? 'text-red-600' : 'text-green-600' }}"></div>
```

同样也要避免使用变量拼接。但可以把完整class作为变量的值：

```jsx
// 不生效
function Button({ color, children }) {
  return (
    <button className={`bg-${color}-600 hover:bg-${color}-500 ...`}>
      {children}
    </button>
  )
}

// 生效
function Button({ color, children }) {
  const colorVariants = {
    blue: 'bg-blue-600 hover:bg-blue-500',
    red: 'bg-red-600 hover:bg-red-500',
  }

  return (
    <button className={`${colorVariants[color]} ...`}>
      {children}
    </button>
  )
}
```

这是因为 `tailwindcss` 的工作原理实际上是使用正则表达式匹配 `class` 名称，因此，只要在代码中使用完整的类名，都可以保证生成正确的、对应的 `CSS`。

#### safelist

对于确实必须使用变量拼接的 `classname`，也可以通过 `safelist` 属性进行显性配置，从而告知 `tailwindcss` 必须应该转换的 `css` 。
可以使用正则表达式的形式，但需要注意这个模式只能匹配基本样式，如果是带修饰符的 `classname`，请使用 variables 列举说明：

```javascript
module.exports = {
  content: [
    './pages/**/*.{html,js}',
    './components/**/*.{html,js}',
  ],
  safelist: [
    'text-2xl',
    'text-3xl',
    {
      pattern: /bg-(red|green|blue)-(100|200|300)/, variants: ['lg', 'hover', 'focus', 'lg:hover'],
    },
  ],
  // ...
}
```

### Theme

`theme` 属性中可以定义项目的颜色、字体、响应式尺寸、边框、间距等样式统一值。
以下是 `tailwindcss` 的默认主题配置：

```javascript
module.exports = {
  theme: {
    screens: {
      sm: '480px', md: '768px', lg: '976px', xl: '1440px',
    },
    colors: {
      'blue': '#1fb6ff', 'purple': '#7e5bef', 'pink': '#ff49db', 'orange': '#ff7849', 'green': '#13ce66', 'yellow': '#ffc82c', 'gray-dark': '#273444', 'gray': '#8492a6', 'gray-light': '#d3dce6',
    },
    fontFamily: {
      sans: ['Graphik', 'sans-serif'], serif: ['Merriweather', 'serif'],
    },
    extend: {
      spacing: {
        '128': '32rem',   '144': '36rem', }, borderRadius: {
        '4xl': '2rem', }
    }
  }
}
```

#### 核心插件
`tailwindcss` 内置了许多可以配置的属性值，以举例 `borderRadius` 的完整定义以及转换后的 `css`：

```javascript
module.exports = {
  theme: {
    borderRadius: {
      'none': '0', 'sm': '.125rem', DEFAULT: '.25rem', 'lg': '.5rem', 'full': '9999px',
    },
  }
}
```

```css
.rounded-none { border-radius: 0 }
.rounded-sm   { border-radius: .125rem }
.rounded      { border-radius: .25rem }
.rounded-lg   { border-radius: .5rem }
.rounded-full { border-radius: 9999px }
```

配置中使用了5个键名：`'none'`、`'sm'`、`DEFAULT`、`'lg'`、`'full'`，这是内置的 `borderRadius` 可以使用的5个后缀名。
注意 `DEFAULT` 的 `className` 并没有对应的后缀，这是所有属性值配置以及使用过程中的约定。

配置之后会覆盖原有的默认值。

对于某一个属性值如果想要完全抛弃默认值，应该在 `corePlugins` 显性标识其失效而不是将其定义为空对象：

```javascript
// 不建议
module.exports = {
  theme: {
    opacity: {},
  }
}

// 建议
module.exports = {
  corePlugins: {
    opacity: false,
  }
}
```

#### 属性值扩展
如果需要保留主题配置的默认属性值，但也想添加新的属性值，可以在 `extend` 中进行扩展配置：

```javascript
module.exports = {
  theme: {
    extend: {
      fontFamily: {
        display: 'Oswald, ui-serif', // Adds a new `font-display` class
      }
    }
  }
}
```

```html
<h1 class="font-display">
  This uses the Oswald font
</h1>
```

也可以自定义修饰语，例如添加一个响应式布局的尺寸值：

```javascript
module.exports = {
  theme: {
    extend: {
      screens: {
        '3xl': '1600px', // Adds a new `3xl:` screen variant
      }
    }
  }
}
```

```html
<blockquote class="text-base md:text-md 3xl:text-lg">
  Oh I gotta get on that internet, I'm late on everything!
</blockquote>
```

#### 引用其他值
如果需要引用主题中的另一个值，可以通过闭包函数并返回属性对象值的方式来实现。闭包将接收一个对象，该对象包含一个 `theme()` 函数，使用该函数即可获取主题中的其他值。

例如，通过在 `backgroundSize` 配置中引用 `theme('spacing')` 进而生成基于 `spacing` 的 `background-size` 属性配置:

```javascript
module.exports = {
  theme: {
    spacing: {
      // ...
    },
    backgroundSize: ({ theme }) => ({
      auto: 'auto', cover: 'cover', contain: 'contain', ...theme('spacing')
    })
  }
}
```

需要注意：该闭包只能对顶级属性使用，而不能对属性配置中的某一个健值使用：

```javascript
// 错误
module.exports = {
  theme: {
    fill: {
      gray: ({ theme }) => theme('colors.gray')
    }
  }
}

// 正确
module.exports = {
  theme: {
    fill: ({ theme }) => ({
      gray: theme('colors.gray')
    })
  }
}
```

### Screens
可以自定义响应式的每一个尺寸值，但需要注意配置的位置：如果是在 `screens` 属性中重新定义，那么没有被定义到的尺寸名称的默认值将会被移除；如果需要保留默认值的基础上对部分尺寸进行重新赋值，应在 `theme.extends.screens` 中进行配置。

```javascript
// 类似于 'xl' 的默认尺寸将会被移除，也就是说：使用 'xl:' 的修饰符是不会被转换的。 
module.exports = {
  theme: {
    screens: {
      'sm': '576px',
      'md': '960px',
      'lg': '1440px'
    },
  }
}

// 原 'xl' 的默认值会被保留，只有 'lg' 对应的值会被覆盖
module.exports = {
  theme: {
    extend: {
      screens: {
        'lg': '992px'
      },
    },
  },
}
```

还可以在 `theme.extends.screens` 增加更多的尺寸：

```javascript
module.exports = {
  theme: {
    extend: {
      screens: {
        '3xl': '1600px',
      },
    },
  }
}
```

属性的值可以是具体的元素值，也可以是一个对象，下面的代码会被正确转换：

```javascript
module.exports = {
  theme: {
    screens: {
      'sm': {'max': '767px'},
      // => @media (max-width: 767px) { ... }

      'md': {'min': '768px', 'max': '1023px'},
      // => @media (min-width: 768px and max-width: 1023px) { ... }

      'lg': {'min': '1024px', 'max': '1279px'},
      // => @media (min-width: 1024px and max-width: 1279px) { ... }

      'xl': {'min': '1280px', 'max': '1535px'},
      // => @media (min-width: 1280px and max-width: 1535px) { ... }

      '2xl': {'min': '1536px'},
      // => @media (min-width: 1536px) { ... }
    }
  }
}
```

### Plugins
插件允许您注册新样式，以便使用 JavaScript 而不是 CSS 将其注入用户的样式表中。

要开始使用你的第一个插件，从 `tailwindcss/plugin` 中导入 `Tailwind` 的插件函数。然后在 `plugins` 数组中，以匿名函数作为第一个参数调用导入的插件函数。

```javascript
const plugin = require('tailwindcss/plugin')

module.exports = {
  plugins: [
    plugin(function({ addUtilities, addComponents, e, config }) {
      // ...
    }),
  ]
}
```

插件函数接收一个对象参数，该参数可以被分解成几个辅助函数:

- `addUtilities()`，用于注册新的静态实用程序样式
- `matchUtilities()`，用于注册新的动态实用程序样式
- `addComponents()`，用于注册新的静态组件样式
- `matchComponents()`，用于注册新的动态组件样式
- `addBase()`，用于注册新的基样式
- `addVariant()`，用于注册自定义静态变量
- `matchVariant()`，用于注册自定义动态变量
- `theme()`，用于在用户的主题配置中查找值
- `config()`，用于查找用户的 Tailwind 配置中的值
- `corePlugins()`，用于检查是否启用了核心插件
- `e()`，用于手动转义在类名中使用的字符串

#### 官方插件
以下插件如需使用应先通过npm安装到项目中，在添加到配置文件中：

```javascript
module.exports = {
  // ...
  plugins: [
    require('@tailwindcss/typography'),
    require('@tailwindcss/forms'),
    require('@tailwindcss/aspect-ratio'),
    require('@tailwindcss/container-queries'),
  ]
}
```
